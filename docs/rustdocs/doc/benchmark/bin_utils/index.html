measure_throughput

Play given TXNs with Benchmarker for several epochs and measure throughput, e.g., the average committed txns per second. Since time is counted from submission until all TXNs are committed, this measurement is in a sense the user-side throughput. Each epoch plays the given TXN pattern sequence repeatedly for several rounds. Return list of summaries for all epochs.
